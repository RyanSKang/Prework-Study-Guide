HTML

  • Hypertext Markup Language (HTML) is one of the 3 cornerstone languages used in web development: HTML, CSS, and JS. HTML is used to create structure of a webpage. Made up of elements which work together to provide content and the basic framework for all pages on the web.
  • HTML elements work together to build the structure of a webpage and provide a framework that makes it easy for a user to navigate the webpage
  • HTML elements are usually composed of an opening tag and a closing tag represented with angle brackets
  • Each HTML element performs a unique role. However, 2 elements are especially important to understanding how the structure works: the head and body elements
    • The head element contains information, or metadata, that helps the browser render the page correctly. Elements included within the head are not visible when a user looks at a webpage
    • The body element contains the content that is shown to the user. All the elements that we want the user to see and interact with must be placed inside this element.
  • !DOCTYPE html declaration: this declaration tells the browser what document type it should expect when it is opened. !DOCTYPE html is letting the browser know to expect HTML on the page. Declarations are not case-sensitive; HTML elements are case sensitive
  • The opening tag represents the root, or start, of our HTML document
  • An HTML attribute defines additional information about an element. Furthermore, HTML attributes are always defined in the beginning element, never the closing element
  • The head element contains the metadata for our web application. Metadata is data about our webpage that the browser uses to effectively serve our webpage to users.
    • Metadata is typically not seen by users, but it contains important information about accessibility, search engines, and performance.
    • Head element is a container for metadata. Meta elements contain information about the page that is used by the browser. The user wont see information in the head or meta elements
    • The title element defines the title for a webpage. This element is required and is extremely important for Search Engine Optimization (SEO).
  • Search engine optimization (SEO) is the process of improving the ranking of a website in search engines such as google or bing. Better SEO results in you website more frequently showing up in searches.
  • When working with visible HTML elements, the content is contained between an opening tag and a closing tag. These tags are most often represented by angle brackets with associated tag names contained inside

Git

  • Github: Web interface for Git, a version control software that allows multiple people to make separate changes to a program at the same time without getting in the way of each other.
  • Terminologies:
    • Repository: Every project on Github is called a repository. A folder for your project that contains all of the files and their revision history
    • Branch: A Workspace where you can make changes that donâ€™t affect the live site.Create a new branch for your work, and name it to best describe the work
    • Markdown (.md): The way the content files are formatted in Github. A way to write for the web using plain text; easily converts to HTML
    • Commit Changes: Saving a new version of a file you are working on. Adds change to your branch
    • Pull Request (PR): Asking for changes on a branch to merge into another branch. A way to discuss and review your work with others
    • Merge: When a Pull Request is approved, changes are "pulled in" or merged into the main branch. Changes are published or deployed to the live site once this happens
    • Issue: Way to report new tasks or content fixes. Track progress on project board, basically a to do list. Open an issue for a new task and close once done!
  • Git Commands:
    • git status: checks out what branch we are currently on
    • git checkout -b branch-name: creates a new branch and switches to it
    • git pull origin main: Allows you to be sure you are working with the most updated information within your branch
    • git push origin main: Allows you to update changes to your current branch
